# 设计决策记录

## 项目目标（探索性）
在 YOLOv26 的 P4、P5 层插入轻量级混合专家模块（SparseDualMoE），探索 MoE 在边缘检测模型上的可行性。**当前仍处于实验阶段，未达成负载均衡目标**。

## 早期设计（数据丢失，仅作背景说明）
在 SparseDualMoE 之前，曾基于 yolo_master 设计了 `UltraOptimizedMoE` 和 `BalancedMoE`（后者增加了异构专家、渐进式 warmup 等）。**因实验数据保存不当，所有量化结果均已丢失**，无法评估其效果。后续直接转向 SparseDualMoE 并进行了多次迭代。

## SparseDualMoE 设计（第三次设计，后续迭代 3+ 次）

### 初始设计（v0.1，对应半月总结阶段 0）
- **结构**：共享专家（1×1 分组卷积）+ 4 个稀疏专家（InvertedResidualExpert，kernel 3/5 交替）+ `UltraEfficientRouter` + 可学习融合门控。
- **路由器**：深度可分离卷积 + 8 倍下采样 + 全局平均池化 + Softmax + Top‑K。
- **动态 Top‑K**：通过复杂度估计器（`AdaptiveAvgPool2d(1) + Conv + Sigmoid`）输出 `complexity`，动态调整激活专家数 `dynamic_top_k = max(1, int(top_k * complexity * capacity_factor))`。
- **辅助损失**：负载均衡损失（硬计数 + detach）+ Z-Loss + 熵正则。

### 发现的问题（阶段 0）
- 深层 MoE（P5）仅专家 3 占 99.9%，浅层 MoE（P4）仅专家 3 占 96.6%，其余死亡。
- 动态 Top‑K 始终为 1（`complexity` ≈0.5，`int(2*0.5*1.5)=1`），每个 token 只激活 1 个专家。
- 负载均衡系数从 0.1588 衰减到 0.0173，后期约束极弱。
- **根因**：硬 Top‑K 梯度阻断 + 负载均衡损失不可微 + 初始化偏置马太效应。

### 迭代改进（阶段 1~5，详细见 TROUBLESHOOTING.md）

| 阶段 | 改进措施 | 结果 | 结论 |
|------|----------|------|------|
| 1 | 强制 `dynamic_top_k = 2`；取消系数衰减 | 深层 2 个专家各 50%，2 个死亡；浅层 2 个活跃 | 硬选择仍导致半数死亡 |
| 2 | 专家 warmup（前 5000 步均匀采样） | 深层 4 个均衡；浅层 3 个（一个濒死<3%） | 浅层偏置更强，需更长 warmup |
| 3 | ε‑greedy（10% 概率随机选专家） | 浅层濒死专家升至 2.17% | 探索率太低 |
| 4 | 线性路由器（无探索） | 坍缩更严重（单专家 99.9%） | 空间信息对路由器重要 |
| 5 | B1（多样化专家 + ε‑greedy 0.2→0.05） | 深层仍为单专家；浅层一个专家消失 | 根本原因未解决 |

## 核心缺陷（当前仍存在）

### 1. 硬 Top‑K 稀疏计算导致梯度阻断
`BatchedExpertComputation` 只对选中的专家前向/反向，非选中专家梯度为 0。即使有负载均衡损失，也只能通过路由器 logits 间接影响，但路由器输出已极度尖锐。

### 2. 负载均衡损失设计缺陷
- `usage` 基于硬计数且 `detach`，梯度无法传递。
- 系数过小（0.1588 → 0.0173），后期无效。

### 3. 专家多样性不足
即使 B1 改进了超参数，所有专家仍共享倒残差结构，输出分布高度相关。路由器只需激活一个专家。

### 4. 初始化偏置与马太效应
初期某个专家因随机原因获得稍高 logit → 被选中 → 获得梯度 → 参数优化 → logit 更高 → 最终垄断。warmup 不足（5000 步）且探索率低（10%），死亡专家无法复活。

### 5. 路由器空间信息丢失（线性路由器实验证实）
卷积路由器可保留部分空间信息，线性路由器完全丢失，坍缩更严重。

## 与理想 MoE 训练机制的差距

| 理想机制 | 当前实现中的实际差距 |
|----------|----------------------|
| Soft Top‑K（所有专家参与前向） | 硬 Top‑K 稀疏计算，非选中专家梯度为 0 |
| 负载均衡损失可微（使用 router_probs） | `usage` 基于硬计数且 `detach` |
| 高负载均衡系数（0.1~0.5）且不衰减 | 系数从 0.1588 衰减到 0.0173 |
| 强制早期均匀采样（前 10%~20% 步数） | 5000 步 warmup（约 10%），探索率仅 0.1 |
| 专家结构显著不同 | 仅 kernel size 交替或超参数差异 |

## 未来改进方向（待验证）
- 改用 **Soft Top‑K**（如加权求和所有专家，或 Gumbel-Softmax）。
- 负载均衡损失使用 **可微的 soft 计数**（直接使用 `router_probs`）。
- 提高负载均衡系数（如 0.5）且不衰减。
- 延长 warmup 至 20%~30% 步数，探索率提升至 0.3~0.5。
- 设计 **显著不同的专家结构**（不同 kernel size、dilation、残差连接）。
- 在 COCO 上训练 300+ epoch 验证。
